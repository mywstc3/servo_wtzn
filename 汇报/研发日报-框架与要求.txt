================================================================================
研发团队工作日报 — 填写要求与框架（全员 / AI 助手共用）
================================================================================

【文件用途】
  后续写日报请 @ 本文件，或说「按 docs/研发日报-框架与要求.txt 写今日日报」。
  AI 根据当日聊天实际内容填写，禁止编造未发生的测试或结果。

【输出文件】
  docs/研发日报-YYYY-MM-DD.txt（UTF-8 纯文本）

【篇幅风格 — 重要】
  参照 docs/研发日报-2026-07-01.txt 的简洁版：
  - 每条工作 2~4 句话，写清做了什么 + 关键数据/结论
  - 不要大段分点、不要重复「操作过程/优化点/进度」子标题
  - 不要附《填写规范》全文；规范只看本文件

================================================================================
填写要求（精简）
================================================================================

1. 内容：写实、有结论；软件写模块/文件，硬件写测点/现象，文档写章节
2. 链接：每条工作末尾必填语雀链接；暂无则写本地 docs 路径 +「待补语雀链接」
3. 卡点：现象 + 已尝试方法 + 阻塞程度 + 是否需支援；无则写「今日工作推进顺畅，无技术、硬件、协作卡点」
4. 协同：P0/P1/P2；无则写「无」
5. 明日计划：1~3 条，带优先级，可落地

================================================================================
AI 信息来源
================================================================================

1. 当日对话中的改代码、排查、测试、文档
2. 上一日 docs/研发日报-*.txt（承接遗留与计划）
3. git 变更（勿虚构烧录结果）
4. 成果链接映射：
   - docs/GD32F1x0-外设笔记.md
   - docs/M-02-M03-感知采集实现笔记.md
   - docs/硬件说明-16999-PS26040802.md
   - docs/功能实现任务拆解-语雀版.md
