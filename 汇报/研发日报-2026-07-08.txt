2、速度/扭矩单位对齐与反馈编码（提交 cb0c079）
纠正 RUN_SPEED：上位机「100」为 100 格/秒（360°=4096 格），内部 °/s = raw×0.087，去掉误用的 RPM×6 导致 100→52.2 的问题。PRESENT_SPEED(0x3A) 改为绝对值；PRESENT_LOAD(0x3C) 实现 encode_load（|duty|×100/3599，负向加 0x0400）。实机测恒速 raw=50 时 plan≈220°/s、实测约 96~98°/s，倾向满占空比物理限速待 Watch output 确认。
成果语雀链接：https://wit-motion.yuque.com/sd8v4v/project_doc/du8lqi1rknyewgxd

4、STS 断连抓包与 I2C 关中断根因修复（提交 6c17373）
解析 READ 0x38 / WRITE 0x29 抓包：断连前校验正常，之后仅有请求无应答。对照 xuniduoji 确认主因是 bsp_i2c_read_reg 整段 __disable_irq，1Mbps 下 USART 丢字节。已去掉 I2C 关总中断；main 多处优先 uart_comm_poll；sts_mem_poll 50ms 刷新反馈、READ 路径降载；ORE 先读 DATA；RX 帧环 10→16；修复 stdbool 与 GD32 bool 冲突。
成果语雀链接：https://wit-motion.yuque.com/sd8v4v/project_doc/du8lqi1rknyewgxd

5、多机超时分析、校准/Flash 试做与 RX 非本机 ID 过滤
单机正常、多机并联易超时；澄清同半双工总线且上位机同时只读选中 ID。成功应答段单机/多机约 50ms 一轮、位置电压正常。曾按参考实现 OFFSET@0x1F、0x0A/0x0B 与 Flash@0x0800FC00，并试 RETURN_DELAY/IRQ 提权/poll_drain，后按要求回到 6c17373 撤销。确认协议无单/多圈模式开关（内部控制用 multi_degree）。现工作区保留未提交的 discard：非本机且非 0xFE 的 ID 按 LENGTH 丢弃整帧。
