## 1. 任务目标

P1 阶段「感知采集」里程碑：固件能稳定读取以下物理量，供后续保护、闭环和通信使用。

| 物理量 | 来源 | 任务编号 |
|--------|------|----------|
| 母线电流 `i_bus` (A) | PA3 ADC ← GS8591 + 0.01Ω shunt | M-02 |
| 母线电压 `v_bus` (V) | PA4 ADC ← VIN 分压 | M-02 |
| 电机位置（角度） | PA9/PA10 I2C ← AS5600 | M-03 |

**验收标准（来自任务文档）：**

| 检查项 | 预期 |
|--------|------|
| 电流 | `i_bus` 可读，趋势与钳形表一致 |
| 电压 | `v_bus` 与万用表误差 < 5%（标定后） |
| 角度 | 旋转时 raw 0~4095 连续变化（除 0/4095 跳变点） |
| 磁铁 | 有磁 `magnet_ok=true`，无磁 `magnet_ok=false` |
| 多圈 | unwrap 后连续正/反转时累计角度单调 |

M-02 与 M-03 **可并行开发**，均不依赖 M-01 完成；但过流保护联调（M-02-7）需要 `motor_disable()` 可用。

## 2. 硬件引脚（原理图 §6、§7、§8 为准）

### 2.1 ADC 采样

| 信号 | MCU 引脚 | 硬件链路 | ADC 通道 |
|------|----------|----------|----------|
| 母线电流 | **PA3** | RS1(0.01Ω) → U7 GS8591 → R11/C15 滤波 | **ADC_IN3** |
| 母线电压 | **PA4** | VIN 经 R1/R4 分压 + D1 钳位 | **ADC_IN4** |

- Shunt：0.01Ω，约 **1 A → 10 mV**
- 运放增益由 R9/R10/R12/R13 决定（BOM 约 100k / 2k 等，**精确增益需按原理图核算或实测标定**）
- 分压比 K_div 由 R1/R4/R15 决定（**文档尚未给出精确阻值，需万用表反推**）

### 2.2 AS5600 磁编码器

| 信号 | MCU 引脚 | 说明 |
|------|----------|------|
| SCL | **PA9** | I2C1_SCL，AF4，R14 上拉 3V3 |
| SDA | **PA10** | I2C1_SDA，AF4，R15 上拉 3V3 |
| I2C 地址 | — | 7-bit **0x36** |
| 角度寄存器 | — | **0x0E**（高字节起），12-bit 有效 |
| 状态寄存器 | — | **0x0B**，bit5 **MD=1** 表示检测到磁铁 |

### 2.3 例程陷阱（必改）

例程 `gpio_output/User/board_config.h` 与本板原理图不一致：

| 项目 | 例程当前 | 本板正确 |
|------|----------|----------|
| AS5600 DIR | **PA4** 配成 GPIO 输出 | **PA4 是 VIN ADC**，DIR 已在硬件接地 |
| I2C 引脚 | PA9/PA10 ✓ | PA9/PA10 ✓ |

移植 `as5600.c` 时：**删除所有 `AS5600_DIR` 相关 GPIO 初始化**，不要在 PA4 上做任何数字输出。

## 3. 代码分层与文件规划

与 M-01 及 [GD32F1x0-外设笔记 §1.2](./GD32F1x0-外设笔记.md) 保持一致：

```text
servo_project/User/
├── board_config.h          # 引脚宏：ADC 通道、I2C 端口（只定义，不写逻辑）
├── servo/servo_config.h    # 标定常数：K_div、运放增益、滤波深度等
├── bsp/
│   ├── bsp_adc.c/h         # [M-02] ADC 时钟/GPIO/单次采样
│   └── bsp_i2c.c/h         # [M-03] I2C1 初始化与寄存器读写
├── servo/
│   ├── adc_monitor.c/h     # [M-02] 换算、滤波、对外 getter
│   └── as5600.c/h          # [M-03] 磁铁检测、角度、unwrap
└── main.c                  # 只调 init / 周期 update，不直接操作寄存器
```

**原则：**

- `bsp_*`：只关心外设初始化与原始读写
- `adc_monitor` / `as5600`：应用语义（安培、伏特、度、多圈）
- 标定系数放 `servo_config.h`，方便实测后只改一处

## 4. M-02 母线电流与电压 — 实现过程

按子任务顺序实施，建议分三阶段：**能读 raw → 能换算 → 能滤波/保护**。

### 4.1 阶段 A：board_config.h 补充宏（M-02-1 前置）

在现有电机 PWM 宏之外增加：

```c
/* 电流 PA3 = ADC_IN3 */
#define ADC_CURRENT_PORT / PIN / CHANNEL

/* 电压 PA4 = ADC_IN4 */
#define ADC_VBUS_PORT / PIN / CHANNEL
```

### 4.2 阶段 B：bsp_adc 初始化（M-02-1 + M-02-2）

**目标：** PA3/PA4 配为模拟输入，ADC 校准后可读到 0~4095 原始值。

**步骤：**

1. **开时钟**
   - `RCU_GPIOA`、`RCU_ADC`
   - `rcu_adc_clock_config(RCU_ADCCK_APB2_DIV6)`（ADC 时钟不超过 14 MHz）

2. **GPIO 模拟模式**
   - `gpio_mode_set(GPIOA, GPIO_MODE_ANALOG, GPIO_PUPD_NONE, PIN_3 | PIN_4)`

3. **ADC 基本配置**
   - `adc_deinit()` → 右对齐 → 规则组长度 1 → 软件触发（不用外部触发）
   - `adc_enable()` 后 **延时若干循环** 再 `adc_calibration_enable()`（GD32 要求）

4. **单通道读取函数** `bsp_adc_read_channel(channel)`
   - `adc_regular_channel_config(0, channel, ADC_SAMPLETIME_55POINT5)`（运放输出有 RC 滤波，采样时间宜长）
   - `adc_software_trigger_enable(ADC_REGULAR_CHANNEL)`
   - 等待 `ADC_FLAG_EOC` → `adc_regular_data_read()` → 清 EOC 标志

**验收：** 万用表测 PA3/PA4 电压同时，Keil Watch 中 raw 随输入变化；空载时两通道读数稳定。

### 4.3 阶段 C：adc_monitor 换算（M-02-4 + M-02-5）

**目标：** raw → 物理量。

在 `servo_config.h` 定义（初值占位，上板标定）：

| 宏 | 含义 |
|----|------|
| `ADC_VREF_V` | 3.3 |
| `ADC_FULL_SCALE` | 4096 |
| `ADC_VBUS_DIV_RATIO` | K_div，分压比 |
| `ADC_CURRENT_R_SHUNT_OHM` | 0.01 |
| `ADC_CURRENT_GAIN` | 运放增益 G |
| `ADC_CURRENT_OFFSET_V` | 零电流时 PA3 电压 |

**公式（与任务文档一致）：**

```text
V_adc = raw × ADC_VREF_V / ADC_FULL_SCALE

v_bus = V_adc × K_div                    /* PA4 */

i_bus = (V_adc - V_offset) / (G × R_shunt)   /* PA3，R_shunt = 0.01 Ω */
```

**标定流程：**

1. **电压：** 用万用表测真实 VIN，读 `v_raw`，令 `K_div = VIN_real / V_adc`
2. **电流：** 空载记录 `V_offset`；加已知负载（或钳形表对比），调整 `G` 使 `i_bus` 一致

**接口设计：**

```c
void adc_monitor_init(void);    /* 调 bsp_adc_init */
void adc_monitor_update(void);  /* 读两路 + 滤波 + 更新内部变量 */
float adc_monitor_get_i_bus(void);
float adc_monitor_get_v_bus(void);
uint16_t adc_monitor_get_i_raw(void);  /* 调试 */
uint16_t adc_monitor_get_v_raw(void);
```

### 4.4 阶段 D：软件滤波（M-02-6）

**目标：** 降低噪声，静态读数稳定。

**推荐：** 8 点滑动平均（`ADC_FILTER_DEPTH = 8`），对电流、电压各维护一个环形缓冲。

- 每帧 `update` 各采一次 PA3、PA4
- 滤波后再做物理量换算（或先换算再滤波，二者择一，建议 **先滤波 raw 再换算**）

### 4.5 阶段 E：定时采样 / DMA（M-02-3，可选进阶）

初版可在主循环或 1~2 ms 定时器中 **轮询** `adc_monitor_update()`，满足 P1 调试。

后续若电流环（M-04）需要 5~10 kHz 同步采样：

- TIMER 触发 ADC 规则组扫描（CH3 + CH4）
- DMA 环形写入双通道缓冲
- ADC 完成或 DMA 半满中断里更新 `i_bus` / `v_bus`

此阶段可后置，不阻塞 P1「可读」里程碑。

### 4.6 阶段 F：过流 / 欠压 / 过压保护（M-02-7）

在 `adc_monitor_update()` 末尾增加阈值判断：

| 条件 | 动作 |
|------|------|
| `i_bus > I_MAX` | `motor_disable()` + 置 fault 标志 |
| `v_bus < V_MIN` | 同上（欠压） |
| `v_bus > V_MAX` | 同上（过压） |

建议加 **迟滞**（例如过流恢复需连续 N 次正常），避免边界抖动。

## 5. M-03 电机位置（AS5600）— 实现过程

### 5.1 阶段 A：board_config.h 补充 I2C 宏（M-03-1 前置）

```c
#define AS5600_I2C          I2C1
#define AS5600_I2C_RCU      RCU_I2C1
#define AS5600_I2C_AF       GPIO_AF_4
#define AS5600_SCL_PORT/PIN  GPIOA / PIN_9
#define AS5600_SDA_PORT/PIN  GPIOA / PIN_10
```

### 5.2 阶段 B：bsp_i2c 初始化（M-03-1）

**步骤：**

1. 开 `RCU_GPIOA`、`RCU_I2C1`
2. PA9/PA10：`gpio_af_set(AF4)` → `GPIO_MODE_AF` → **开漏** + 内部上拉（板上有 R14/R15 外部上拉，内部上拉可选）
3. `i2c_deinit(I2C1)` → `i2c_clock_config(400000)` → 7-bit 主机模式 → `i2c_enable`

**底层读写（参考例程）：**

- `bsp_i2c_write_reg(addr, reg, data, len)` — 写寄存器
- `bsp_i2c_read_reg(addr, reg, data, len)` — 先写寄存器地址，再 repeated start 读
- 每步等 I2C 标志位，加 **超时**（如 10 ms）防死锁

**验收：** 能读 AS5600 `0x0B` STATUS，返回值不是 0xFF/全失败。

### 5.3 阶段 C：as5600 驱动（M-03-2 + M-03-3）

**寄存器：**

| 寄存器 | 地址 | 用途 |
|--------|------|------|
| STATUS | 0x0B | bit5 MD = 磁铁检测 |
| ANGLE  | 0x0E | 读 2 字节，取低 12 bit |

**接口：**

```c
void as5600_init(void);              /* 调 bsp_i2c_init */
bool as5600_is_connected(void);      /* 读 STATUS 是否 I2C 成功 */
bool as5600_magnet_detected(void);   /* STATUS & 0x20 */
uint16_t as5600_read_angle_raw(void); /* 0 ~ 4095 */
float as5600_read_angle_deg(void);   /* raw × 360 / 4096 */
```

**注意：** 不要初始化 PA4 为 DIR 输出。

### 5.4 阶段 D：角度 unwrap（M-03-4）

AS5600 单圈 12-bit，过 0°/360° 边界时 raw 会 **4095→0 跳变**。

**算法：**

```text
delta = raw - raw_prev
if delta > 2048:  delta -= 4096
if delta < -2048: delta += 4096
angle_unwrapped += delta
raw_prev = raw
```

**接口：**

```c
void as5600_update(void);              /* 读 raw 并更新 unwrap 内部状态 */
int32_t as5600_read_angle_unwrap(void); /* 多圈累计 counts */
```

### 5.5 阶段 E：速度估算（M-03-5，P1 可选）

```text
speed = Δangle_unwrapped / Δt
```

- 在固定周期（如 2 ms）调用 `as5600_update()`
- 速度可再经一阶低通平滑
- P1 若只验证位置，可跳过；速度环（M-05）前必须完成

### 5.6 阶段 F：无磁铁安全（M-03-6）

- `as5600_magnet_detected() == false` 时：
  - 禁止 `motor_enable()` 或闭环使用角度
  - 可选：置 fault、LED 指示
- 主循环或 init 中先检测磁铁再进入控制

## 6. 集成到工程

### 6.1 init_all 扩展

在 `servo/Init.c` 的 `init_all()` 中，于 `motor_init()` 之后增加：

```c
adc_monitor_init();
as5600_init();
```

顺序建议：systick → gpio → motor → adc → as5600（无强依赖，ADC 与 I2C 可互换）。

### 6.2 main 最小验证循环

**不要**在验证感知阶段默认 `motor_enable()`。

```c
while (1) {
    adc_monitor_update();
    as5600_update();
    /* 调试：Watch 窗口观察 getter，或 LED 指示 magnet_ok */
    delay_1ms(2);
}
```

在 `project/template.uvprojx` 的 **bsp** 组添加：

- `bsp_adc.c`、`bsp_i2c.c`

- `adc_monitor.c`、`as5600.c`

库文件 `gd32f1x0_adc.c`、`gd32f1x0_i2c.c` 已在 Library 组，无需重复添加。

## 7. 调试与排错

| 现象 | 排查 |
|------|------|
| ADC 读数全 0 或全 4095 | PA3/PA4 是否 `GPIO_MODE_ANALOG`；是否做了校准；VDDA 是否 3V3 |
| 电压明显偏差 | 标定 `K_div`；D1 钳位后有效范围 |
| 电流零点漂移 | 重新测 `ADC_CURRENT_OFFSET_V`（空载） |
| I2C 无应答 | PA9/PA10 是否 AF4；AS5600 是否供电（U8）；上拉是否存在 |
| 角度不变化 | 磁铁是否安装、气隙 0.5~3 mm；`magnet_detected` 是否为 true |
| 角度跳变 | 正常现象，需 unwrap；单圈显示用 `read_angle_deg` |
| 例程 as5600 编译冲突 | 删除 PA4 DIR 相关代码 |
