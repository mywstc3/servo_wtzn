# 速度观测改进方案

> **用途**：解决 1000 Hz 差分速度在 AS5600 12-bit 量化下出现 0↔88 °/s 抖动的问题。  
> **关联文件**：`speed.c/h`、`encoder.c`、`time.c`、`main.c`  
> **编写日期**：2026-07-09  
> **状态**：🔶 **工作区已实施「MA(4) + 跨距差分 SPAN=16 + 真实 Δt」**（覆盖 S-03；以跨距差分替代单步）。**VOFA 验收待做**。旁支 `feature/speed-ls-observation` @ `aaa0a61` 另有完整 S-01 线性回归可对比。

---

## 0. 问题背景

### 0.1 现象

VOFA+ 观测到 `motor_speed_degree`（ch2）在 `plan_speed ≈ 50 °/s` 时大幅振荡，峰值约 **±88 °/s**，波形呈锯齿/准正弦，而 `plan_speed`（ch3）仍为平稳直线。

### 0.2 根因

| 原因 | 说明 |
|------|------|
| **量化 + 单步差分** | AS5600 1 LSB ≈ 0.0879°；1000 Hz 下相邻采样 +1 格 → `speed_raw ≈ 88 °/s` |
| **固定 Δt 假设** | `speed_raw = Δθ × MOTOR_SPEED_CALC_HZ`，未用真实采样间隔 |
| **滤波窗口偏短** | 中值(5) + MA(8) + LPF(0.3) 对 0/88 交替抑制不足 |
| **主循环抖动** | I2C 阻塞导致 `speed_update` 间隔非严格 1 ms |

### 0.3 数据流（当前工作区 vs 旧）

**旧（`4c15bb9` 及更早）：**

```text
encoder_update()  1000 Hz  →  motor_encoder_degree（12-bit 量化）
speed_update()    1000 Hz  →  Δθ × 1000 → （曾用）中值/MA → LPF
```

**当前工作区 `feature/traj-stribeck-flash` WIP：**

```text
raw unwrap → accum_raw → multi_degree
  → MA(SPEED_MA_WIN=4)
  → 环形缓冲 filt_angle[] / filt_tick[]（g_control_tick）
  → speed = (filt[newest]-filt[oldest]) / Δt_sec   （SPAN=16，约 16 ms）
  → LPF α=0.3 → motor_speed_degree
```

**旁支 `feature/speed-ls-observation`：**

```text
多圈角 → ls_angle[]/ls_tick[]（N=20）→ 最小二乘斜率 → 弱 LPF(α=0.05)
```

### 0.4 选型原则

| 优先级 | 目标 |
|--------|------|
| P0 | 消除 0↔88 °/s 量化尖峰 |
| P1 | 控制滞后 < 20 ms（速度环 500 Hz） |
| P2 | 不改硬件、不改 STS 协议 |

### 0.5 实施结论（2026-07-14）

| 项 | 状态 |
|----|------|
| S-03 真实 Δt | ✅ 工作区已用 `g_control_tick` |
| S-01 线性回归 | 🔶 旁支已实现；主分支工作区未合入 |
| 跨距差分（等效降噪） | ✅ 工作区 `SPEED_DIFF_SPAN=16` + 前置 MA(4) |
| VOFA §9 验收 | ❌ 待实机录波确认 |

若跨距方案验收不达标，再对比合入旁支 LS；二者勿混拷结构体字段。

## 2. S-01 多点线性回归速度（首选）

### 2.1 原理

用最近 N 个 `(t_i, θ_i)` 做最小二乘直线拟合，斜率即速度：

```text
ω = (N·Σ(tθ) − Σt·Σθ) / (N·Σ(t²) − (Σt)²)
```

等间隔且 `t_i = i·Δt` 时可简化为：

```text
ω ≈ (N·Σ(i·θ_i) − (Σi)(Σθ)) / (Δt · (N·Σ(i²) − (Σi)²))
```

量化噪声随 N 增大约按 **1/√N** 下降，优于单步差分。

### 2.2 参数建议（1000 Hz 观测）

| 参数 | 建议值 | 说明 |
|------|--------|------|
| N | **16** | 16 ms 窗口，滞后约 8 ms |
| 备选 N | 10 / 20 | 更小 → 噪声大；更大 → 滞后大 |
| 后续滤波 | 可选保留弱 LPF | 回归后可去掉中值/MA，仅 LPF α=0.1 |

### 2.3 数据结构（`speed.h`）

```c
#define SPEED_LS_WIN    16U

typedef struct {
    /* ... 现有字段 ... */
    float ls_angle[SPEED_LS_WIN];   /* 多圈角样本 (°) */
    uint32_t ls_tick[SPEED_LS_WIN]; /* g_control_tick 时间戳 */
    uint8_t ls_idx;
    uint8_t ls_count;
} speed_observer_t;
```

### 2.4 实现要点（`speed.c`）

```c
#include "time.h"

static float speed_ls_slope(speed_observer_t *obs)
{
    /* 用 ls_angle[] 与 ls_tick[] 计算最小二乘斜率，返回 deg/s */
}

void speed_update(speed_observer_t *observer, sensor_data_t *sensor)
{
    /* unwrap + 多圈角（现有逻辑） */
    /* push motor_angle_multi_degree + g_control_tick 进环形缓冲 */
    if (observer->ls_count < SPEED_LS_WIN) {
        return;
    }
    speed_raw = speed_ls_slope(observer);
    /* 可选：中值 → MA → LPF，或仅 LPF */
}
```

### 2.5 等间隔简化版

若 S-03 保证 Δt 稳定，可预计算 `Σi`、`Σi²` 系数表，每次只更新环形缓冲求 `Σθ`、`Σ(i·θ)`，减少运行时除法。

### 2.6 验收

- 50 °/s 匀速：`motor_speed_degree` 峰峰值 < **5 °/s**（原约 88）
- 阶跃响应：相对 `plan_speed` 滞后 < **20 ms**

## 3. S-02 加大平滑窗口

### 3.1 参数建议

| 参数 | 当前 | 建议 |
|------|------|------|
| `SPEED_MEDIAN_WIN` | 5 | **9** 或 **11** |
| `SPEED_MA_WIN` | 8 | **32** 或 **64** |
| `SPEED_LPF_ALPHA` | 0.3 | **0.05 ~ 0.1** |

### 3.2 改动

仅修改 `speed.h` 宏，Rebuild 烧录。

### 3.3 优缺点

- ✅ 改动最小，可立刻验证
- ❌ 滞后 30~60 ms，速度环易跟不紧
- ❌ 不能从根本消除 88 °/s 尖峰，仅压低幅值

### 3.4 适用场景

S-01 实施前的 **临时验证**；或与 S-01 叠加（回归输出后仅保留弱 LPF）。

## 4. S-03 真实 Δt（推荐与 S-01 联用）

### 4.1 问题

当前代码：

```c
speed_raw = diff_degree * (float)MOTOR_SPEED_CALC_HZ;  /* 假定严格 1 ms */
```

主循环 + I2C 阻塞时，实际 `speed_update` 间隔可能是 0.7~2 ms，仍乘 1000 会导致速度系统性偏大/偏小。

### 4.2 改法

```c
#include "time.h"

static uint32_t s_last_speed_tick;

/* speed_update 内 */
uint32_t now = g_control_tick;
uint32_t dt_ticks = now - s_last_speed_tick;
s_last_speed_tick = now;

if (dt_ticks == 0U) {
    return;
}
float dt_s = (float)dt_ticks / (float)CONTROL_LOOP_HZ;
speed_raw = diff_degree / dt_s;   /* °/s */
```

线性回归方案中，**必须**在 `ls_tick[]` 存真实时间戳，回归斜率单位才正确。

### 4.3 注意

- `g_control_tick` 为 `volatile uint32_t`，单次读取即可
- 首帧或 `dt_ticks == 0` 时跳过，避免除零
- 与 S-01 联用时不要用固定 `MOTOR_SPEED_CALC_HZ` 乘子代替 Δt

## 5. S-04 观测频率降至 500 Hz

### 5.1 改法

```c
// encoder.h
#define ENCODER_UPDATE_HZ    500U
```

`time.c` 分频 `1000 / 500 = 2`，每 2 ms 采一次；`MOTOR_SPEED_CALC_HZ` 自动变为 500。

### 5.2 效果

- 1 LSB 尖峰：88 → **~44 °/s**
- 观测 : 速度 PID = 500 : 500（1:1，边界可接受）
- I2C 负载减半，主循环更易实时

### 5.3 优缺点

- ✅ 改动小，立刻减噪
- ❌ 未根治量化问题；有效观测带宽降低
- 若坚持「观测频率 > 速度环」，应保留 1000 Hz 观测 + 500 Hz PID，用 **S-01** 而非单纯降频

## 6. S-05 小 diff 死区

### 6.1 改法

```c
#define SPEED_DIFF_LSB    (360.0f / 4096.0f)
#define SPEED_DIFF_DEAD   (0.5f * SPEED_DIFF_LSB)

diff_degree = /* 现有多圈角差分 */;
if (diff_degree > -SPEED_DIFF_DEAD && diff_degree < SPEED_DIFF_DEAD) {
    diff_degree = 0.0f;
}
speed_raw = diff_degree / dt_s;
```

### 6.2 适用

- 静止、极低速时抑制抖动
- **不能**单独解决 50 °/s 工况（diff 通常大于 0.5 LSB）

### 6.3 定位

作为 S-01 / S-03 的 **辅助**，非主方案。

## 7. S-06 AS5600 RAW / 多次平均

### 7.1 现状

`encoder.c` 连读 5 字节（STATUS + RAW + ANGLE），速度计算用的是 **ANGLE（0x0E，芯片内滤波）**，未单独使用 RAW（0x0C）。

### 7.2 选项

| 选项 | 做法 | 说明 |
|------|------|------|
| A | 改读 RAW ANGLE | 响应快，噪声可能更大，需更强软件滤波 |
| B | 保持 ANGLE | 芯片已滤波，当前问题主要在软件差分 |
| C | 同周期读 2 次取平均 | I2C 负载翻倍，不推荐 @1000 Hz |
| D | 降低读频、软件插值 | 与 1000 Hz 观测目标矛盾 |

### 7.3 建议

**优先 S-01**，暂不改 AS5600 寄存器选择。可做对比试验：VOFA 增加 `motor_encoder_raw` 通道，观察静止时是否在 ±1 格抖动。

## 附录 A · 量化噪声速查

| 观测率 | 1 LSB 差分对应 speed_raw |
|--------|---------------------------|
| 100 Hz | ~8.8 °/s |
| 500 Hz | ~44 °/s |
| 1000 Hz | **~88 °/s** |

真实速度 ω（°/s）每样本理论格数：

```text
格/样本 = ω / (360/4096) / 观测_Hz
```

例：50 °/s @ 1000 Hz → 约 0.57 格/样本 → 读数在 0/1 格间抖 → 差分速度大幅跳动。
