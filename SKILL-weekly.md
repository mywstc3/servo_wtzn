# 研发团队周报 PPT 生成器

## 目的

根据用户每天口述的工作内容（口语化描述、关键词、要点即可），在周末自动汇总生成一份格式规范的周报 PPT，风格与团队现有模板一致。

## 调用时机

- 用户要求"生成周报"、"写周报"、"周报PPT"、"weekly report"
- 用户说"帮我汇总本周工作"、"生成本周工作总结"
- 用户在周末想生成一份 PPT 汇报本周工作
- 用户说"记录今日工作"（在周报上下文中，会累积到本周数据中）

## 输出规范

### 输出文件

生成一个 `.pptx` 文件到当前工作目录下，文件命名格式为 `周报_MM.DD-MM.DD.pptx`（如 `周报_7.7-7.11.pptx`）。

### PPT 结构（3 页）

| 页码 | 标题 | 内容 |
|------|------|------|
| Slide 1 | 本周工作 {日期范围} | 本周完成的重点工作项，可含子条目（具体做了什么、结果） |
| Slide 2 | 下周计划 {日期范围} | 下周计划开展的工作项 |
| Slide 3 | 长期任务 / 其他事项 | 持续进行的项目、长期任务、备注等 |

### 风格规范

- 幻灯片尺寸：16:9 宽屏
- 标题：44pt 加粗，深灰蓝色（#44546A）
- 主条目：18pt 加粗，蓝色强调（#4874CB），带 `▎` 前缀
- 子条目：16pt，灰色（#595659），缩进
- 字体：微软雅黑

## 执行流程

### 方式一：用户口述本周工作总结（推荐）

用户一次性描述本周工作情况（可口语化），AI 提取关键信息后生成。

**步骤 1：收集信息**

询问用户：
- 本周日期范围（默认：本周一到周五）
- 本周完成了哪些工作？（每项做了什么、有什么结果）
- 下周计划做什么？
- 有没有长期在做的任务？

**步骤 2：提取并分类**

将用户口述内容归类到三个板块：
- **本周工作**：提取已完成的工作项，每项写清楚具体内容和结果
- **下周计划**：提取计划中的工作项
- **长期任务**：提取持续进行中的长期项目

用户口述可能不完整，需要追问：
- "本周大概做了几件事？每件能再展开说说吗？"
- "下周主要是推进哪些方向？"
- "长期在做的项目有哪些？"

**步骤 3：生成 JSON 数据**

将整理好的内容保存为临时 JSON 文件，格式如下：

```json
{
  "date_range": "7.7-7.11",
  "this_week": [
    {
      "text": "PDR算法优化",
      "link": "https://wit-motion.yuque.com/sd8v4v/project_doc/za8is2g4u5lure9r",
      "sub": ["完成5组测试", "准确率从85%提升到92%", "代码解耦重构"]
    },
    {
      "text": "台球杆测试",
      "link": "https://wit-motion.yuque.com/sd8v4v/project_doc/dytf4uxpabba8mwc",
      "sub": ["出杆偏移已修复", "击球力量需继续优化"]
    }
  ],
  "next_week": [
    {"text": "羽毛球挥拍指标开发", "sub": ["输出速度、力量指标"]}
  ],
  "long_term": [
    {"text": "触觉传感器研发"},
    {"text": "振动上位机算法"}
  ]
}
```

**语雀链接处理规则：**
- 如果日报中有语雀文档链接（`yuque.com` / `wit-motion.yuque.com`），**必须**提取并填入对应条目的 `link` 字段
- 主条目有 `link` → PPT 中该行末尾显示可点击的蓝色下划线 "📎语雀文档"
- 子条目有 `link` → 子条目末尾显示 "📎链接"（子条目也支持 `{"text": "...", "link": "..."}` 格式）
- 一个条目可以有多个子条目各自带不同的链接
- 没有链接的条目省略 `link` 字段即可

**步骤 4：调用生成脚本**

```bash
python generate_weekly_ppt.py \
  --json <临时JSON文件路径> \
  --output "周报_<日期范围>.pptx"
```

**步骤 5：输出结果**

向用户展示：
- 生成的文件路径
- PPT 三页的内容摘要
- 提醒用户用 PowerPoint/WPS 打开检查

### 方式二：从每日日报汇总

如果用户本周已通过 `daily-work-log` skill 生成了日报 `.md` 文件：

1. 询问用户日报文件存放的目录（如未指定，则在当前工作目录下递归搜索文件名含"日报"或"工作日报"的文件，不限格式：`.md`、`.txt`、`.csv` 等）
2. 按本周日期范围筛选匹配的日报文件（文件名中包含本周日期即视为匹配）
3. 读取每份日报，提取关键内容：
   - 从"二、今日完成工作"提取 → 归入「本周工作」
   - 从"六、明日工作计划"提取 → 归入「下周计划」
   - 从"七、其他备注"提取 → 归入「长期任务」
   - **从日报中提取所有语雀链接**（`yuque.com` / `wit-motion.yuque.com`），匹配到对应的工作条目
4. 去重、合并同类项
5. 按方式一的步骤 3-5 生成 PPT

### 方式三：每日累积记录

如果用户想每天记录、周末一键生成：

1. 用户说"记录今日工作" → AI 将今天的内容追加到本周的累积 JSON 文件中
2. JSON 文件路径：`./_weekly_cache_{年份}{周号}.json`
3. 周末用户说"生成周报" → AI 读取累积文件 → 按方式一的步骤 3-5 生成 PPT

累积 JSON 格式：
```json
{
  "week": "2026-W28",
  "days": {
    "周一": {
      "done": [{"text": "...", "sub": ["..."]}],
      "plan": ["..."],
      "issues": ["..."]
    },
    ...
  },
  "long_term": [{"text": "..."}]
}
```

## 内容整理原则

参考日报 skill 的规范，周报整理时遵循：

1. **禁止一句话简写**：每项工作写清楚做了什么、过程、结果。禁止出现"调算法"这种模糊描述。
2. **适度扩充**：用户口述可能是零散的，AI 负责整理为正式表述，但**不编造具体数据**。
3. **工作类型适配**：
   - 硬件调试类 → 补充测试场景、迭代次数、验证结果
   - 软件/算法类 → 补充迭代版本、测试用例、实验数据
   - 文档/需求类 → 补充撰写内容、完成度
4. **去重合并**：同一项目/模块的工作合并为一个主条目，不同时间的推进作为子条目。
5. **关键结果优先**：本周工作的每条必须体现明确的结果或进度节点。
6. **子条目控制**：每个主条目下最多 3-4 条子条目。如果子条目太多，考虑拆分主条目。
7. **语雀链接必附**：日报中有语雀链接的，周报中必须对应附上。链接使得 PPT 中的 "📎语雀文档" 标签可点击跳转，方便汇报时直接打开查看详情。

## 交互原则

- 首次使用时询问用户姓名，后续自动填充到备注
- 用户提供的信息越少，追问越精确，不编造事实
- 如果用户同时提到了昨日遗留和今日计划，自动判断归属
- 生成的 PPT 允许用户反馈"XXX 需要修改"，AI 直接修改 JSON 后重新生成
- 如果用户没有提到长期任务 → 标题改为"其他事项"，内容填"本周暂无"

## 示例

### 用户输入（口语化）

> 这周主要是调PDR算法，跑了大概5组测试吧，准确率现在到92%了。然后去惠州测了一天台球杆，出杆偏移的问题修好了，但是击球力量还有点问题。另外调研了一下乒乓球的视觉识别方案，初步定了用YOLO加双目相机。下周打算开始搞羽毛球的挥拍指标，还有要帮触觉传感器那边做测试。

### AI 处理

1. 提取归类：
   - 本周工作 3 项（PDR算法优化、台球杆测试、乒乓球视觉识别调研）
   - 下周计划 2 项（羽毛球挥拍指标、触觉传感器测试）
2. 追问："有没有长期在做的项目？比如一直在迭代的那种？"
3. 生成 JSON → 调用脚本 → 返回 PPT 路径

```

  ▎PDR算法优化  📎语雀文档    ← 蓝色下划线，点击跳转语雀
```

## 注意事项

- 生成的临时 JSON 文件在 PPT 生成后保留，方便用户后续修改和重新生成
- 如果 `python-pptx` 未安装，提示用户运行 `pip install python-pptx`
- PPT 生成脚本 `generate_weekly_ppt.py` 需放在当前工作目录下（或确保在 PATH 中可找到）
- Windows 下注意路径分隔符用 `/` 或 `\\`
